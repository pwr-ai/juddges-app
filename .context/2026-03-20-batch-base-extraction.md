# Batch Base-Schema Extraction (Issue #37) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add checkpoint/resume, skipped-tracking, and completion reporting to the existing batch extraction script so it meets all acceptance criteria for the v1 demo corpus.

**Architecture:** Extend `scripts/run_base_extraction.py` with a local JSONL checkpoint file that records per-document outcomes. After the run, generate a JSON completion report. Add `--checkpoint` flag for resume and `--report` flag for report path.

**Tech Stack:** Python 3.12+, asyncio, supabase-py, loguru, JSONL for checkpoints

---

### Task 1: Add checkpoint/resume to the extraction script

**Files:**
- Modify: `scripts/run_base_extraction.py`

**What:** Add a local JSONL checkpoint file (`data/extraction_checkpoint_<timestamp>.jsonl`) that records `{doc_id, status, jurisdiction, timestamp, error}` after each document. On resume (`--checkpoint <path>`), load checkpoint, skip already-completed IDs, and continue.

**Step 1: Add checkpoint helper functions**

Add three functions:
- `load_checkpoint(path)` - reads JSONL, returns set of completed doc IDs
- `append_checkpoint(path, doc_id, status, jurisdiction, error)` - appends one line
- `generate_report(checkpoint_path, report_path, args, elapsed)` - produces JSON summary

**Step 2: Add CLI args `--checkpoint` and `--report`**

- `--checkpoint PATH` - path to existing checkpoint for resume (optional)
- `--report PATH` - path to write completion report (default: `data/extraction_report_<timestamp>.json`)

**Step 3: Wire checkpoint into extraction loop**

After each document result, append to checkpoint. Before processing, filter out IDs already in checkpoint (skipped count). Track skipped alongside completed/failed.

**Step 4: Generate completion report at end of run**

JSON report with: timestamp, model, jurisdiction_filter, total_found, completed, failed, skipped, elapsed_seconds, breakdown_by_jurisdiction.

**Step 5: Run syntax check**

Run: `python -m py_compile scripts/run_base_extraction.py`
Expected: No output (success)

**Step 6: Run help to verify new args**

Run: `python scripts/run_base_extraction.py --help`
Expected: Shows --checkpoint and --report args

**Step 7: Commit**

```bash

git commit -m "feat(extraction): add checkpoint/resume and completion report to batch script"
```
