### Key Findings

1. **43% of non-adversarial queries return zero results** — the search system fails to find any documents for nearly half of meaningful queries, including common patterns like "contract breach damages" and "Supreme Court conviction overturned".
2. **No fuzzy/typo tolerance** — all misspelled queries (3/3) return zero results. The `pg_trgm` extension is enabled but only indexes `case_number`, not content fields.
3. **Polish full-text search is broken** — no Polish stemmer/dictionary is configured. Polish queries use `'simple'` regconfig which provides no morphological analysis. Polish words like "odpowiedzialność" (liability) won't match inflected forms.
4. **Vector/semantic search is unavailable** — all queries with `alpha > 0` fail with Internal Server Error. The embedding model service is not running or misconfigured.
5. **High latency** — pure text search averages 1073ms (p95=1509ms), far above the 500ms target for interactive search.
6. **Score scale mismatch in hybrid fusion** — vector scores (0-1) and ts_rank scores (unbounded floats) are combined with linear weighting, producing unreliable ranking when both are active.

## Recommended Improvements (Prioritized)

### Critical (implement immediately)

1. **Fix vector/semantic search (embedding service)**
   - Impact: Enables hybrid search, expected to double relevance scores
   - Effort: Configuration fix (1-2 hours)
   - Files: `.env`, `backend/app/embedding_providers.py`, Docker Compose
   - The embedding service must be running for `alpha > 0` to work. Currently all hybrid/vector searches fail with 500 error.

2. **Add Polish full-text search dictionary**
   - Impact: Fix 33% zero-result rate for Polish queries, enable Polish stemming
   - Effort: 2-4 hours (SQL migration + testing)
   - Files: New Supabase migration, `search_judgments_hybrid` function
   - Install `ispell` or `hunspell` Polish dictionary, create `'polish'` text search configuration, update search function to use language-aware config per document.

3. **Implement Reciprocal Rank Fusion (RRF) instead of linear combination**
   - Impact: Fix ranking inconsistency between vector (0-1) and text (unbounded) scores
   - Effort: 4-6 hours (SQL function rewrite)
   - Files: `supabase/migrations/` (new migration)
   - Formula: `RRF_score = sum(1 / (k + rank_i))` where k=60 is standard. This normalizes scores across different retrieval methods.

### High Priority

4. **Add fuzzy/typo-tolerant search on content fields**
   - Impact: Fix 100% misspelled query failure rate
   - Effort: 4-8 hours
   - Files: New migration for `pg_trgm` indexes, update search function
   - Add trigram GIN indexes on `title`, `summary` columns. Use `similarity()` or `%` operator for fuzzy matching as a fallback when FTS returns zero results.

5. **Reduce search latency baseline**
   - Impact: Achieve <500ms p50 target
   - Effort: 2-4 hours investigation
   - Files: Backend connection pooling, Supabase config
   - Investigate: Is the 900ms baseline from connection overhead? Consider connection pooling, prepared statements, or moving to direct PostgreSQL connection instead of Supabase REST API.

6. **Set `ef_search` parameter for HNSW index**
   - Impact: Improve vector search recall at query time
   - Effort: 30 minutes
   - Files: New migration or session-level SET
   - `SET hnsw.ef_search = 100;` (default is typically 40, higher = better recall but slower)

### Medium Priority

7. **Add `websearch_to_tsquery` as query parser alternative**
   - Impact: Better handling of natural language queries, boolean operators
   - Effort: 2-3 hours
   - Files: `search_judgments_hybrid` SQL function
   - `websearch_to_tsquery` handles quoted phrases, OR, - (NOT), and is more forgiving than `plainto_tsquery`.

8. **Improve snippet extraction**
   - Impact: Fix 1.71/5 snippet quality score
   - Effort: 4-6 hours
   - Files: `search_judgments_hybrid` SQL function
   - Currently returns first 500 chars of summary/full_text. Should use `ts_headline()` to extract passages containing the search terms with highlighting.

### Low Priority / Future

9. **Add cross-encoder reranking** for top-K results
   - Impact: Improve ranking quality for hybrid search
   - Effort: 1-2 days
   - Use a cross-encoder model (e.g., `cross-encoder/ms-marco-MiniLM-L-6-v2`) to rerank top 20-50 results

10. **Implement query classification/routing**
    - Impact: Route citation lookups to exact match, concept queries to vector search
    - Effort: 1 day
    - Detect query type (case number, statute reference, concept) and adjust alpha automatically
