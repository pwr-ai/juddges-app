Business Context

Objective

Provide legal researchers and practitioners with semantic search across court judgments from Poland and the United Kingdom, combining full-text retrieval with AI-powered relevance ranking.

Target Users

  • Legal researchers and academics
  • Lawyers and law firms
  • Policy analysts and government bodies
  • Law students studying comparative law

Key Value Propositions

  • Cross-jurisdictional search (PL + UK in one platform)
  • Semantic understanding of legal concepts
  • RAG-based chat for legal Q&A
  • Structured metadata (judges, dates, courts, keywords)

Data Sources

  • Polish: HFforLegal/case-law (~50K+ cases, sampling ~3K)
  • UK: JuDDGES/en-appealcourt (6,154 judgments)
  • Embeddings: OpenAI text-embedding-3-small (768d)
  • Reranking: Cohere rerank-v3.5

Technology Stack

Frontend

  • Next.js 15 (App Router)
  • React 19 + Radix UI
  • Tailwind CSS 4
  • Zustand (global state)
  • React Query (server state)
  • TipTap (rich text editor)
  • Playwright + Jest (testing)

Backend

  • FastAPI (Python 3.12+)
  • LangChain / LangGraph
  • OpenAI API (GPT-4o, embed-3-small)
  • Cohere (rerank-v3.5)
  • Celery + Redis (task queue)
  • Langfuse (observability)
  • Ruff + pytest (quality)

Data & Infrastructure

  • Supabase (PostgreSQL + Auth)
  • pgvector (HNSW, 768d)
  • Docker Compose (dev + prod)
  • Docker Hub (image registry)
  • GitHub Actions (CI/CD)
  • HuggingFace Hub (datasets)
  • Semver git tags (versioning)
